版本: v6-disk-scan-and-resfilter
日期: 2026-05-02
特性: 🎞 改为扫磁盘列视频 + register 分辨率过滤可控

关键改动:
  - 后端新增 /auto_sequential/list_dir_videos 接口
      直接扫 output_dir 磁盘 (不依赖会话队列), 返回所有视频
      每条带分辨率 / mtime / size / in_queue 标记
      max_count 上限 200, since_timestamp 可选过滤
  - 后端 /auto_sequential/register_video 加 only_target_res 参数
      默认 true (兼容旧行为, 推荐): 只登记 target_resolution 的视频, 过滤掉预览版
      false: 取 since_timestamp 之后最新的视频, 不管分辨率
      失败时返回 skipped_reason = "no_target_res_match", 前端可识别为预期跳过
  - 节点新增 widget register_only_target_res (BOOLEAN, 默认开)
      控制上述参数, 用户可关掉过滤
  - 前端 🎞 按钮改名 "选视频抽尾帧（从 output_dir）"
      改为调 list_dir_videos 接口, 列出磁盘上所有视频
      弹窗工具栏: 「只看 target_resolution」复选框 + 文件名搜索框
      每条标注: 文件名 / 分辨率 (★target / 非 target) / 时间 / 大小 / 是否在队列
      响应式过滤, 队列里的视频带 📜 标记
  - 前端 register_video 失败 with skipped_reason = "no_target_res_match"
      console.log (而非 warn), 提示用户: 可关掉 register_only_target_res

向后兼容:
  - 所有旧按钮、旧接口行为不变
  - register_video 默认开过滤 = 旧行为
  - 老工作流加载时 register_only_target_res 用默认值 true

修复的根因:
  原版 🎞 只看会话队列 (list_videos), 队列依赖 execution_success 触发的 register。
  在多次工作流之间, 或预览版/终版同时输出导致 register 跳过的情况下, 队列会
  停留在旧值 (例: 磁盘到 18, 队列只到 16)。新版直接扫磁盘, 永远反映真实状态。
